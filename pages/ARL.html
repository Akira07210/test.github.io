Automatique Request Luna

Objectif de l'application : Certains des appareils que j'ai pu tester sont supervisés par serveur web. Mon objectif a donc été d'automatiser les actions des appareils en passant par la plateforme et de traiter les résultats remontés.

⚠️

Pour des raisons de secret professionnel, le code source et le fonctionnement exacte de ce programme ne seront pas partagé.
Merci de votre compréhention.

1. Fonctionnement Général

Mon application (nommée ARL) va s'authentifier sur le serveur pour ensuite monitorer les appareils voulu. Une fois que les produits en tests ont fini d'exécuter les taches demandés, l'ARL se reconnecte à la plateforme pour récupérer les résultats. Les résultats sont ensuite triés, puis stocké de manère claire dans un fichier log pour que le testeur puisse facilement voir les informations importantes.

2. Découpage et Analyse des Composants

A l'aide de l'interface graphique, l'utilisateur peut en toute simplicité renseigner les identifiants des produits à tester. Ces identifiants sont stockés, et seront réutilisés dans les requêtes avec le serveur.

Pour communiquer avec la plateforme web j'utilise les biliothèques BeautifulSoup pour analyser simplement les réponses web et request pour communiquer avec le serveur.

Pour chaque requête faite, l'ARL prend en compte les réponses http des requêtes, car le serveur n'est pas toujours joignable. Si une requête n'obtient pas un code 200 ou 201 alors l'ARL le renseigne dans les logs et attend avant de retenter sa requête.

Après avoir attendu un délai fixé par l'utilisateur pour laissé le temps au produit d'effectuer sa tache, l'ARL refait une connexion plateforme pour récupérer les résultats de la demande. Seul les informations en rapport avec le test sont récoltés.

Les informations les plus importantes comme le résultat de la requête sont directement communiqués avec l'interface graphique. Cela permet de voir rapidement l'état du test.
Toutes les informations en rapport avec le test sont enregistrés dans un fichier log.csv. Si un doute est émit sur les résultats obtenu, il est toujours possible d'aller vérifier le résultat manuellement avec l'id de la requête stocké dans les logs(gestion d'erreur).

Pour garentir un fonctionnement continue du programme l'utilisation de la méthode try{} exept{} est très régulière. Elle permet de traiter les cas d'erreur et ne pas intérromptre le programme. Les cas erreur sont également inscrits dans l'interface graphique.
